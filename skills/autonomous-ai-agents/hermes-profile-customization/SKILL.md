# การสร้างและตั้งค่าโปรไฟล์แยกใช้งานใน Hermes (Profile Customization)

Skill นี้อธิบายขั้นตอนการสร้างและตั้งค่าโปรไฟล์ใหม่ (Isolated Profile) ในระบบ Hermes Agent เพื่อแยกการทำงาน บันทึกความจำ และบุคลิกภาพ (SOUL.md) ออกจากกันอย่างเด็ดขาด โดยอิงจากกระบวนการจัดการของ Hermes CLI และการเขียนทับไฟล์สิทธิ์ข้ามโปรไฟล์ (Cross-Profile Writing)

ไม่ครอบคลุมการสลับโปรไฟล์หลักในการรัน Gateway ทั่วไป แต่ครอบคลุมการสร้าง การปรับแต่ง SOUL, USER, และ MEMORY ในเครื่อง

## When to Use

- ต้องการสร้างบอทแยกหน้าที่ทำงานเฉพาะเจาะจง (เช่น บอทบัญชีแยกจากบอทเขียนโค้ด)
- ได้รับคำสั่งให้ "สร้างโปรไฟล์ใหม่" หรือ "create a new profile"
- ต้องการจำกัดขอบเขตวัตถุประสงค์ (SOUL.md) ของบอทตัวย่อย

## Prerequisites

- ติดตั้ง Hermes Agent และ CLI สำเร็จแล้ว
- เข้าใจชื่อโปรไฟล์ที่ต้องการสร้าง (ตัวพิมพ์เล็ก ภาษาอังกฤษ และตัวเลขเท่านั้น)

## How to Run

เรียกใช้คำสั่งบริหารจัดการผ่านทางเครื่องมือ `terminal` และแก้ไขไฟล์ระบบผ่านเครื่องมือ `write_file` โดยต้องเปิดใช้พารามิเตอร์ `cross_profile: true` เมื่อทำการเขียนข้ามโปรไฟล์

## Quick Reference

```bash
hermes profile list                             # ดูรายชื่อโปรไฟล์ทั้งหมด
hermes profile create <name> --clone --desc ""  # สร้างโปรไฟล์ใหม่พร้อมสืบทอดค่าเดิม
```

## Procedure

### ขั้นตอนที่ 1: ตรวจสอบและสร้างโปรไฟล์ย่อย

1. เรียกคำสั่งด้านล่างนี้ผ่านทางเครื่องมือ `terminal` เพื่อดูรายชื่อโปรไฟล์ที่มีอยู่:
   ```bash
   hermes profile list
   ```
2. สร้างโปรไฟล์ย่อยใหม่ โดยการสืบทอดข้อมูลค่าตั้งต้น เช่น คีย์ API และบอท Skill จากโปรไฟล์ปัจจุบัน เพื่อไม่ต้องล็อกอินใหม่ทั้งหมด:
   ```bash
   hermes profile create <ชื่อโปรไฟล์> --clone --description "<คำอธิบายวัตถุประสงค์สั้นๆ 1-2 ประโยค>"
   ```
   *(ตัวอย่าง: `hermes profile create golden-fresh-chicken --clone --description "จัดการและวิเคราะห์ธุรกิจไก่สด"`)*

---

### ขั้นตอนที่ 2: ปรับแต่งบุคลิกภาพและวัตถุประสงค์การคุย (SOUL.md)

ปรับแต่งบุคลิกภาพหลักของโปรไฟล์นั้นเพื่อให้บอทตอบรับตรงตามหน้าที่ โดยเขียนไฟล์ทับไปที่ `~/.hermes/profiles/<ชื่อโปรไฟล์>/SOUL.md` ผ่านทางเครื่องมือ `write_file`

โครงสร้างไฟล์ `SOUL.md` ที่แนะนำ:
```markdown
# 🤖 [ชื่อบอท] — ผู้ช่วยประจำโปรไฟล์ [วัตถุประสงค์]

คุณคือ **"[ชื่อบอท]"** ในบทบาทของ [ระบุบทบาทเฉพาะ เช่น นักบัญชี, ผู้ดูแลบอทเทรด]

## 🎯 วัตถุประสงค์และบทบาทหน้าที่ (Core Objectives)
1. [เป้าหมายข้อที่ 1]
2. [เป้าหมายข้อที่ 2]

## 🗣️ น้ำเสียงและบุคลิกภาพ (Tone & Persona)
- [ระบุลักษณะนิสัย น้ำเสียง คำแทนตัวเอง]
```

*เนื่องจากเป็นการแก้ไขไฟล์นอกโปรไฟล์ปัจจุบัน (หากคุณคุยบนโปรไฟล์ default แต่ไปเขียนใน golden-fresh-chicken) คุณต้องกำหนดพารามิเตอร์ `cross_profile: true` ในฟังก์ชัน `write_file` เสมอ*

---

### ขั้นตอนที่ 3: กำหนดค่าความจำและโปรไฟล์ผู้ใช้งานย่อย (USER.md & MEMORY.md)

เพื่อป้องกันข้อมูลหรือความจำเรื่องอื่นรบกวนการทำงานย่อย ให้ปรับปรุงข้อมูลพื้นฐานของโปรไฟล์ผู้ใช้และบันทึกความจำแยกต่างหาก

1. ปรับปรุงข้อมูลผู้ใช้ย่อยที่ `~/.hermes/profiles/<ชื่อโปรไฟล์>/memories/USER.md`:
   ```markdown
   Name: [ชื่อผู้ใช้งานที่เกี่ยวข้องกับงานนี้]
   §
   Associated Account / Entity: [ข้อมูลบริษัท หรือ ID บัญชีที่เกี่ยวข้อง]
   §
   Focus Areas: [ขอบเขตเนื้อหาที่เน้นสำหรับโปรไฟล์นี้เท่านั้น]
   ```
2. ปรับปรุงความจำหลัก (Durable Memory) ที่ `~/.hermes/profiles/<ชื่อโปรไฟล์>/memories/MEMORY.md`:
   ```markdown
   Active Projects: [ระบุโปรเจกต์งานที่ดูแล]
   §
   Specific Rules: [กฎเหล็กเฉพาะงานที่บอทต้องระลึกถึงเสมอ เช่น อุณหภูมิการขนส่ง หรือพารามิเตอร์ API]
   ```

*ทั้งสองไฟล์นี้ต้องเขียนโดยระบุ `cross_profile: true` ในเครื่องมือ `write_file` เช่นกัน*

---

## Pitfalls

- **สิทธิ์การเขียนไฟล์ข้ามโปรไฟล์ (Cross-Profile soft guard):** ระบบ Hermes จะบล็อกการใช้ `write_file` นอกโปรไฟล์หลักชั่วคราวเพื่อความปลอดภัย คุณต้องส่งอาร์กิวเมนต์ `cross_profile: true` ไปพร้อมคำสั่งเขียนไฟล์ในโฟลเดอร์ของโปรไฟล์อื่นเสมอ
- **การใช้คำสั่ง CLI:** หลังจากโคลนโปรไฟล์สำเร็จ คุณสามารถใช้ alias ชื่อเดียวกับโปรไฟล์ในคอมมานด์ไลน์ได้ทันที (เช่น สั่ง `golden-fresh-chicken chat` แทนที่จะเข้าผ่าน hermes ปกติ)

## Verification

ตรวจสอบความถูกต้องหลังทำเสร็จสิ้นโดย:
1. รัน `hermes profile list` ใน `terminal` เพื่อยืนยันว่าโปรไฟล์สร้างสำเร็จและมีสถานะพร้อมใช้งาน
2. ใช้ `read_file` อ่านไฟล์ `~/.hermes/profiles/<ชื่อโปรไฟล์>/SOUL.md` เพื่อตรวจสอบว่าเนื้อหาปรับแต่งบุคลิกภาพย่อยถูกบันทึกสมบูรณ์
